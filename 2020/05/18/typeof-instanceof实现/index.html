前言

typeof和instanceof常用来判断一个变量是什么类型的。本篇会说明两者的区别并手写一个instanceof

typeof

  • 对于typeof来说，只返回以下几个结果：number、boolean、string、function、object、undefined

  • 注意function这种引用类型是会返回function的，并不会返回Object

  • 对于其他的引用类型参数，时终返回object。这一点导致无法判断准确的类型

  • 有一个很特殊的值，null

    1. null不是一个空引用, 而是一个原始值, 参考ECMAScript5.1中文版 4.3.11节; 它只是期望此处将引用一个对象, 注意是”期望”, 参考 null - JavaScript.
    2. typeof null结果是object, 这是个历史遗留bug, 参考 typeof - JavaScript
    3. 在ECMA6中, 曾经有提案为历史平凡, 将type null的值纠正为null, 但最后提案被拒了. 理由是历史遗留代码太多, 不想得罪人, 不如继续将错就错当和事老, 参考 harmony:typeof_null [ES Wiki]

  • typeof 来获取一个变量是否存在，如 if(typeof a!=”undefined”){alert(“ok”)}，而不要去使用 if(a), 因为如果 a 不存在（未声明）则会出错。

    instanceof

  • instanceof是用来判断一个变量是否属于某个对象的实例

  • 官方的话：该运算符用来测试一个对象在其原型链中是否存在一个构造函数prototype属性

  • 这个判断比较准确哦！
